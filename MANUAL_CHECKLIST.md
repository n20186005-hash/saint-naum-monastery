# 网站手动检查清单

> 在部署前，请按照此清单逐项检查，确保网站在各个环境下均能稳定运行。

## 🔧 自动化检查

在开始手动检查前，先运行自动化检查脚本：

```bash
# 安装依赖（如果未安装）
npm install

# 运行所有检查
npm run check:all

# 或单独运行各项检查
npm run check:json      # 验证 JSON 文件格式
npm run check:links     # 检查链接和图片资源
npm run check:build     # 运行构建检查（需较长时间）
```

- [ ] **iPad（768x1024）**

  - 单列布局，无水平滚动
  - 文字大小合适，易于阅读

## 4. 🔗 链接检查

### 4.1 内部链接
逐页检查以下链接：
- [ ] **Logo/站点名称** → 返回首页
- [ ] **导航菜单链接** → 跳转到正确页面
- [ ] **页脚链接**（隐私政策、服务条款等） → 跳转到正确页面
- [ ] **语言切换器** → 正确切换中英文，URL 路径正确

### 4.2 外部链接
- [ ] **Google Maps 链接** → 在新标签页中打开 Google Maps
- [ ] **社交媒体链接**（如有） → 打开正确的社交媒体页面
- [ ] **外部资源链接**（如有） → 打开正确的外部页面

检查方法：
1. 鼠标悬停在链接上，查看浏览器底部状态栏显示的 URL 是否正确
2. 点击链接，验证跳转目标是否正确
3. 对于外部链接，验证是否在新标签页打开（`target="_blank"`）

## 7. ⚡ 性能检查

### 7.1 使用 Lighthouse 检查（Chrome DevTools）
1. 打开 Chrome 开发者工具（F12）
2. 切换到 **Lighthouse** 面板
3. 选择 **Performance**（性能）、**Accessibility**（无障碍）、**Best Practices**（最佳实践）、**SEO**
4. 点击 **Generate report**

目标分数：
- [ ] **Performance**（性能）≥ 80
- [ ] **Accessibility**（无障碍）≥ 90
- [ ] **Best Practices**（最佳实践）≥ 80
- [ ] **SEO** ≥ 80

### 7.2 页面加载速度
- [ ] 首屏内容（Hero 组件）在 1 秒内显示
- [ ] 整个页面在 3 秒内完全加载（在快速 3G 网络条件下测试）

测试方法：
1. 打开 Chrome 开发者工具（F12）
2. 切换到 **Network** 面板
3. 模拟慢速网络：**Network** → **No throttling** → 选择 **Slow 3G**
4. 刷新页面，观察加载时间

## 🛠️ 常见问题及解决方案

### 问题 1：图片不显示
**可能原因**：
- 图片文件路径错误
- 图片文件不存在
- 服务器未正确配置静态文件服务

**解决方案**：
1. 检查图片文件路径是否正确（区分大小写）
2. 确认图片文件存在于 `public/` 目录中
3. 运行 `npm run check:links` 自动检查

### 问题 2：样式错乱
**可能原因**：
- CSS 文件未正确加载
- Tailwind CSS 配置错误
- 浏览器缓存问题

**解决方案**：
1. 清除浏览器缓存（Ctrl+Shift+R 或 Cmd+Shift+R）
2. 检查 CSS 文件是否正确导入
3. 重新构建项目（`npm run build`）

### 问题 3：链接 404 错误
**可能原因**：
- 链接路径错误
- 页面文件不存在
- Next.js 路由配置错误

**解决方案**：
1. 检查链接路径是否正确
2. 确认目标页面文件存在于 `src/app/` 目录中
3. 检查 `src/i18n/routing.ts` 中的路由配置
